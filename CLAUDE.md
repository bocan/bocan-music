## Build & development commands

```bash
make bootstrap        # one-time setup: brew deps + fpcalc + git hooks
make generate         # regenerate Bocan.xcodeproj from project.yml (run after project.yml changes)
make build            # debug build
make test             # Xcode unit tests (view models + observability; no snapshots)
make test-coverage    # tests + coverage report (fails if < 80%)
make test-ui          # UI SPM package: snapshot + view-model tests
make test-audio-engine  # AudioEngine SPM tests (requires FFmpeg via Homebrew)
make test-persistence # Persistence SPM tests
make test-metadata    # Metadata SPM tests
make test-library     # Library SPM tests
make test-acoustics   # Acoustics SPM tests
make lint             # SwiftLint + SwiftFormat lint
make format           # auto-format all Swift files
make doctor           # verify tool versions
```

**Never hand-edit `.pbxproj`.** All project structure changes go in `project.yml`, then `make generate`.

## Architecture overview

Bòcan is a native macOS music player (Swift 6, SwiftUI, macOS 26+). The project is an **Xcode project generated by XcodeGen**, with all logic in **seven independent Swift Package modules** under `Modules/`.

### Module dependency order (bottom → top)

```
Observability → Persistence → AudioEngine → Metadata → Library → Playback → UI → App
```

No module imports one above it in this stack. Only the `UI` module may import `AppKit`.

| Module | Responsibility |
|--------|---------------|
| `Observability` | `AppLogger` facade, MetricKit, telemetry. Never use `print` or raw `os_log`. |
| `AudioEngine` | `AudioEngine` actor, `EngineGraph`, `BufferPump`, FFmpeg bridge for OGG/Opus/APE/DSD |
| `Persistence` | GRDB 7 `Database` actor, migrations (M000–M014+), repositories, `AsyncThrowingStream` observation |
| `Metadata` | TagLib bridge (`TagReader`/`TagWriter`), `LRCParser`, `CoverArtExtractor` |
| `Library` | `LibraryScanner`, FSEvents watcher, security-scoped bookmark helpers |
| `Playback` | `PlaybackQueue` actor, `QueuePlayer`, `GaplessScheduler`, `CrossfadeScheduler`, sleep timer |
| `UI` | SwiftUI views, `LibraryViewModel`, `NowPlayingViewModel`, `TracksViewModel`, `AlbumsViewModel` |

### App entry point (`App/BocanApp.swift`)

`BocanApp` owns every subsystem and constructs them once at launch: `Database`, `AudioEngine`, `QueuePlayer`, `LibraryScanner`, `LyricsService`, and all ViewModels. These are injected down via SwiftUI environment or direct init — not created inside views. The root UI is `BocanRootView` (a `NavigationSplitView` with sidebar, content pane, and `LyricsPane`).

### Gapless playback

`GaplessScheduler` polls every 500 ms. When remaining time ≤ 7 s it calls `AudioEngine.enableGaplessNext(url:)`, which pre-decodes the next track into ring buffers. If the format (sample rate + channel count) matches the current track, the next track is scheduled on the **same** `AVAudioPlayerNode` using nanosecond `AVAudioTime` anchoring — no gap. Mismatched formats fall back to stop/load/play (~30–100 ms gap).

### Database observation pattern

`Database` exposes changes as `AsyncThrowingStream<T>` via GRDB `ValueObservation`. Observers emit the current value immediately and on every change. ViewModels consume these streams via `Task { for try await value in db.observe(...) { ... } }`.

### Test split: Xcode vs SPM

`make test` runs the `BocanTests` Xcode target (no host app, no AppKit rendering — view model tests only). Snapshot tests live in the `UI` Swift package and run via `make test-ui`. Never put snapshot tests in the Xcode target.

## Coding standards

These apply to all new code; they override any defaults.

**Concurrency (Swift 6 strict)**
- Long-lived mutable state → `actor`, not a class with locks.
- View state → `@MainActor`.
- No `DispatchQueue.global().async` — use `Task` / `TaskGroup`.
- Always check `Task.checkCancellation()` in loops over `AsyncSequence`.
- `@preconcurrency` is allowed only at documented third-party boundaries (AVFoundation). Add a `// TODO: remove when <framework> adds Sendable` comment.

**Error handling**
- Each module defines one `*Error: Error, Sendable` enum with context-carrying cases.
- No bare `try?` without an `else { log.warning(...) }`.
- `fatalError` is banned outside `#if DEBUG` or a genuinely unreachable `default:`.

**Logging** — use `AppLogger` from `Observability`:
- Log `debug` at start and end of every async op (include elapsed ms on end).
- Log `error` for every caught error (include `String(reflecting: err)`).
- Redact keys matching `Observability.sensitiveKeys` (`apiKey`, `token`, `password`, …).

**Testing** — Swift Testing (`import Testing`, `@Test`, `#expect`, `#require`):
- 80% line coverage minimum per module (CI-enforced).
- Every bug fix starts with a failing regression test.
- No network calls in tests — use `URLProtocol` stubs or protocol mocks.
- Snapshot tests: light + dark mode, representative sizes.

**Commits** — Conventional Commits with module scope:
`feat(audio):`, `fix(ui):`, `chore(persistence):`, etc.

**Accessibility**: every interactive element needs `accessibilityLabel`; full keyboard navigation required.

**Localization**: use String Catalogs (`.xcstrings`); no string literals in views.

## FFmpeg & fpcalc

`AudioEngine` links against Homebrew FFmpeg (system module map, Option B). Build outside Xcode:

```bash
cd Modules/AudioEngine
PKG_CONFIG_PATH=/opt/homebrew/opt/ffmpeg/lib/pkgconfig swift build
```

`fpcalc` (Chromaprint) and its ~15 FFmpeg dylibs are bundled in `Resources/` with paths rewritten to `@loader_path`. They are **not in git**. Re-bundle after a Homebrew FFmpeg/Chromaprint upgrade:

```bash
make bundle-fpcalc   # copies + relinks dylibs
make generate        # only needed if dylib filenames changed
```
